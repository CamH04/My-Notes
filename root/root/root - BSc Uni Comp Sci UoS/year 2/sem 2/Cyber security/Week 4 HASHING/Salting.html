Salting

Create a random string to be added to the password
Hash both the random string and the password
, We get a unique hash for each user, even if they use the same password

 

 

Well, now we need to hash the salt don’t we? The attacker can find the hashed password and the salt and generate a new password list

Sure, the attacker can do that – but now they need a new rainbow table or hash table for EVERY user
That becomes a lot of work
