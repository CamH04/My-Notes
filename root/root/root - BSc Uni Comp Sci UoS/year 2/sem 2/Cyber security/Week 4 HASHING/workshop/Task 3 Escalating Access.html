Task 3 Escalating Access

The previous stage we should have found some logon credentials for the database. These are likely an individual users credentials, and wont have access to the administration options we want, or all the data we want. So lets fix that.

On kali, open a terminal window and type

mysql -h 192.168.100.60 -u wordpress -p

Enter the password we found.

Type:

SHOW DATABASES;

Have a look at what we have found. The tzab3 database looks like it may contain some interesting information, and at the very least we should have some access to the wordpress database.

Type:

use tzab3; 

Did it work? Why not? Type:

use wordpress;

Type:

select * from wp_users;

This is interesting, there is an admin username and password. Get both. The password has been hashed, but this is not a big deal. Hashing can be undone. Particularly if it is a simple password. One method is to create a list of possible passwords (a dictionary) and hash them using the same method as has been used by the database.
